# Correspondence flow

Incoming correspondence moves through registration, executive resolution, execution, approval, EDS signature, outgoing numbering, dispatch and closure. Registration requests a number through a repository; page components do not generate official numbers.

The v1.0.1 working slice covers registration, duplicate inspection, registry display, case context, send-for-resolution, task claim/completion and audit history. Later slices extend the same aggregate with resolution editing, response authoring, signature and dispatch commands.
